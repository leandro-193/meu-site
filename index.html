Engenharia de Software & Metodologia Scrum

O que é Engenharia de Software:

A Engenharia de Software é uma disciplina da engenharia que se concentra em todos os aspectos da produção de software, desde as fases iniciais de especificação do sistema até a manutenção desse sistema após sua entrada em operação.

O que é Scrum:

Scrum é um framework ágil para gerenciamento e planejamento de projetos de software. No Scrum, os projetos são''' divididos em ciclos chamados de Sprints, que representam um período dentro do qual um conjunto de atividades deve ser executado.

Objetivos deste trabalho

  • Compreender os fundamentos da Engenharia de Software
  • Explorar a metodologia ágil Scrum
  • Entender como aplicar esses conceitos em projetos reais

Fundamentos de Engenharia de Software

A base para construir sistemas de software de qualidade, escaláveis e manuteníveis.

Processos de Software

Um processo de software é um conjunto de atividades relacionadas que levam à produção de um produto de software.

Modelos de Processo

Existem diversos modelos de processo de desenvolvimento de software, como o Modelo Cascata, Incremental, Espiral, entre outros.

Qualidade de Software

Garantir que o software atenda às necessidades do usuário com qualidade, considerando confiabilidade, usabilidade, eficiência e manutenibilidade.

Metodologia Scrum

Um framework ágil que ajuda equipes a trabalhar de forma colaborativa e entregar valor continuamente.

Product Owner

Maximiza o valor do produto e do trabalho do time de desenvolvimento.

Scrum Master

Facilita o processo, remove impedimentos e mantém a equipe alinhada às práticas ágeis.

Time de Desenvolvimento

Profissionais responsáveis por entregar incrementos utilizáveis.

Eventos principais

  • Sprint (2‑4 semanas)
  • Sprint Planning
  • Daily Scrum
  • Sprint Review
  • Sprint Retrospective
